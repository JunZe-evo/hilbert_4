总体框架（为什么他的方式有效）

核心：问题驱动 → 公理化 → 教与研并重 → 长期题库

方法归纳为 8 个可操作原则（适用于综合学科），并配套练习与检验标准。

1. 把问题写清楚（问题驱动）

一句话写问题 + 已知 / 未知 / 限制条件。

检验：5 行内写出本质并给出两个子问题。

2. 建体系（公理化／抽象化）

抽象“最小假设 + 基本定义”，追溯结论依赖。

3. 阅读并重写（不满足“会读”）

把论证重写成可复述的结构（两种结构更佳）。

4. 教学即学习（讲给别人听）

20 分钟讲稿 / 录音 / 讲义，讲结构不讲堆砌。

5. 个人题库（长期题目仓）

分级（可解 / 待研究 / 开放），周期回看。

6. 批判性讨论

质疑关键假设，尝试反例 / 替代方案。

7. 存在性 vs 构造性

写清类型，对存在性尝试构造化。

8. 跨域迁移

把工具迁移到新领域小问题里验证。

实践清单

  1. 边界明确：一句话 + 已知 / 未知 / 限制。
  2. 抽象公理化：最小定义 + 基本假设 + 必要性检验。
  3. 重写结构：定义—中间结论—主结论—主线。
  4. 教学输出：20 分钟讲稿 / 录音 / 讲义。
  5. 长期题库：分级 + 周期回看。
  6. 批判讨论：反例 / 替代假设。
  7. 存在 vs 构造：类型标注 + 构造化尝试。
  8. 跨域迁移：工具搬迁 + 小问题验证。
